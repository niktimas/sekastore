# Промпт для следующей итерации

Ты работаешь в папке проекта `C:\Users\niktimas\Downloads\seka`. Пользователь является официальным дистрибьютором велосипедов SEKA и хочет сайт для доменов `seka-bike.ru` и `seka-bike.store`. VPS будет на Ubuntu 24.04, база данных PostgreSQL, реализация на Node.js.

Перед началом работ прочитай:

- `docs/ARCHITECTURE.md`
- `docs/PRODUCT_MODEL.md`
- `docs/CONTENT_AND_UX.md`
- `docs/ROADMAP.md`

Исходные материалы:

- Папки моделей: `seka spear standart`, `seka spear rdc`, `seka exceed standart`, `seka exceed rdc`, `seka exaero GR`.
- В каждой папке есть `описание.txt` и изображения цветов.
- `система.png` содержит данные по системам и колесам.
- `наличие 13.07.jpg` содержит наличие на 13.07.2026.

Работай по следующим принципам:

1. Сначала сохраняй и нормализуй реальные данные из локальных файлов, не выдумывай характеристики, цены или наличие.
2. Для названий в публичном интерфейсе используй корректное `Standard`, даже если папки названы `standart`.
3. Делай сайт как премиальный каталог официального дистрибьютора, а не как шаблонный маркетинговый лендинг.
4. Frontend должен быть SEO-friendly, поэтому предпочтителен Next.js.
5. Backend и работа с данными должны быть на Node.js/TypeScript.
6. База данных - PostgreSQL, миграции через Prisma или другой выбранный ORM, если это явно обосновано.
7. Наличие, системы и колеса из картинок сначала нужно оцифровать и показать пользователю на проверку перед записью в базу.
8. Не коммить `.env`, создай `.env.example`.
